# 治理机制

## 概述

WRAITH 协议的治理完全由链上持有者驱动。无团队席位，无否决权，无特殊角色。

## 提案流程

```
1. 提案创建
   └── 需持有 ≥ 100,000 $WRAITH

2. 讨论期
   └── 72 小时链上讨论窗口

3. 投票期
   └── 48 小时，按持仓量加权

4. 执行
   └── 通过阈值: >50% 赞成 + ≥ 10% 总供应量参与
```

## 可治理参数

- AI 成长基金使用方向
- 社区奖励分发规则
- 协议升级提案
- 跨链扩展计划

## 进入 Phase 03 后

当协议达到 Phase 03（具现），治理系统全面上线：
- 链上投票合约部署
- 治理界面开放
- 多签执行机制激活
